# 💸 勾配割り勘ツール (Gradient Bill Splitting Tool)

本リポジトリは、React + TypeScript + Tailwind CSS で構築されたシングルページアプリケーション（SPA）「勾配割り勘ツール」です。

## 📝 概要

「勾配割り勘ツール」は、旅行や飲み会、イベント後の精算をスマートに行うためのWebアプリです。
単なる均等割りだけでなく、メンバーごとの負担割合（勾配）を設定でき、さらに端数を誰が負担するかまで指定できます。
最終的には、送金回数が少なくなるように「誰が誰にいくら払うか」を自動算出します。

## 🎯 目的

- 不公平感の少ない精算を実現する
- 支払いフロー（送金回数）を最適化する
- 計算過程を可視化し、透明性を担保する

## ✨ 機能

### 👥 メンバー・イベント管理

- イベント名の設定
- 参加メンバーの追加・削除（名前のみで登録）

### 🧾 精算項目管理

- 項目名、金額、立替者の登録
- 対象メンバーのオン/オフ選択
- 精算項目の編集・削除

### 🎛️ 勾配（負担割合）設定

- 各メンバーの割合（%）を個別設定
- あるメンバーの比率を編集すると、未編集メンバーへ残り比率を自動再配分
- 勾配モードOFF時は対象者で均等割り

### ✂️ 端数管理

- 各精算項目で、切り捨て計算後に生じる端数の負担者を指定可能
- 指定した端数負担者が無効な場合は、対象者から自動で補正

### 📊 精算結果・明細表示

- 貪欲法（Greedy Algorithm）による最小送金回数の精算結果表示
- メンバー別に、項目ごとの計算式・負担額・支払済額・収支を表示

### 💾 データ管理

- JSON保存/読込（イベント状態を丸ごと保存・再開）
- CSVエクスポート（明細を表計算ソフトで共有可能）

## 🧮 内部計算アルゴリズム

各メンバーの基本負担額は、以下の式に基づき切り捨てで計算されます。

$$
B_{e,m} = \left\lfloor C_e \times \frac{w_{e,m}}{\sum_{k \in M}w_{e,k}} \right\rfloor
$$

- $B_{e,m}$: メンバー $m$ の項目 $e$ における基本負担額
- $C_e$: 精算項目 $e$ の金額
- $w_{e,m}$: メンバー $m$ の重み（勾配モード時は入力％、均等割り時は 1）
- $\sum_{k \in M}w_{e,k}$: 対象メンバーの重み合計

切り捨てで生じた端数は端数負担者へ加算し、全メンバーの収支をもとに貪欲法で送金フローを導出します。
